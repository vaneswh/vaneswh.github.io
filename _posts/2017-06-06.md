title: wordpress sql迁移至bitcron

###所有不了了之总归需要一个完结
在转投Farbox之前，我在wordpress写了近五年的blog。彼时有一段心理低潮期，莫名地停掉了这种记录方式，域名和虚拟主机到期后也就没再续费。
过了大半年，偶然在围脖上看到Farbox的介绍，心下一动，重新开了个实验品XD 断断续续用到现在，跟随着产品迭代到Bitcron，恍恍惚惚又快五年过去。
当时wordpress备份的sql，至今默默地躺在硬盘里。受限于自己的所知，拿它束手无策。
但是所有的债终有要还的一天...
###耗时整整一天的迁移
逛淘宝时，无意看到如今竟然有虚拟主机可以按月售卖，且价格很便宜。突然想到也许能利用这个把sql恢复，利用wp自带的exporter导出成xml再作处理。
随便找了一家巴黎云主机，带cpanel面板的美国主机一个月仅需6RMB ：）好久没有进行过类似数据库操作，感觉生疏不少。借助伟大的google、度娘，好不容易恢复成功。过程简易记录如下：
1. 虚拟主机绑定域名（这个供应商的产品有点奇葩，不绑定域名无法访问），新建A记录指向主机ip；
2. 安装wordpress3.5版本（话说wp已经更新到4.x了，但是新旧版本的差别还是比较大的，所以我找到了当时的旧版本，防止不兼容）；
3. 打开phpmyadmin，找到对应数据库，删除其中所有表（11个），导入备份的sql（导入后也是11个表，废话>_<!），由于当时用的域名和现在的域名并不一样，所以还得更新下各种链接。我使用的[方法](http://jingyan.baidu.com/article/066074d68e9361c3c31cb04f.html)是在phpmyadmin里直接运行下列语句：
```
UPDATE wp_options SET option_value = REPLACE(option_value,'旧地址','新地址');
UPDATE wp_posts SET post_content = replace(post_content,'旧地址','新地址');
UPDATE wp_posts SET post_excerpt= replace(post_excerpt,'旧地址','新地址');
UPDATE wp_posts SET guid= replace(guid, '旧地址','新地址');
```
4. 到此为止，刷新下首页，会出现wp更新数据库的提示，点击确认后，blog恢复就算完成。
5. 利用wp自带导出器，导出xml文件。
但是如何把xml转成markdown，更让人抓耳挠腮。现有成熟的解决方案：
1. Bitcron设置中自带的由wp导入，难度☆☆☆☆☆
2. Farbox上原来的[在线工具](http://wp2md.farbox.com/)，难度☆☆☆☆☆
3. 通过wordpress插件export to jekyll，难度★☆☆☆☆
4. 通过github上各路开发者的包，以exitwp最有名，但是往往需要用到Python，难度★★★☆☆
5. 其他野路子，暂时没摸清...
经过本人不懈尝试，发现这几个主流办法都没成功，简直是被逼进了死胡同。至于失败的原因，至今也没搞懂。第一，Bitcron的导入基本失败，只导入了posts的标题，且日期均不是原文撰写日期而是导入日期；第二，Farbox的工具可以转换，生成的md以wp下的category为文件夹，文件名为原标题拼音，但遇到标题含特殊字符时，会有问题；第三，插件可以顺利安装，但是使用时会报错，错误google不出来，不知是哪路bug；第四，作为一枚彻头彻尾的Python门外汉，用pip install死活安装不了git，报错“Command "python setup.py egg_info" failed with error code 1 in”。stackeroverflow上有人说是管理员权限的问题，但是最后折腾几番也没能成功解决。
山穷水尽疑无路之时，一篇blog让我发现了新大陆，github上有个大神写了个可执行程序：[wpXml2Jekyll](https://github.com/theaob/wpXml2Jekyll)，简直神器。仅仅12.5kb，界面上就两个按钮：选择xml文件、保存文档。但是效果杠杠滴，我试了试，wp中的页面和文章会全部被导入一个文件夹，文件名为wp里设置的slug，postmeta信息也基本被正确转换成md中的head信息，内容中涉及的编排格式也大致无误。好像md也支持html的语法？
