##python语法基础
* 变量名，标识符（允许作为变量名或其他对象名称的有效符号）命名规则：（受C语言影响）

 * 首字符必须为字母或下划线
 * 其余可以是字母、下划线、数字
 * 大小写敏感
 * 好的标识符应该做到以下几点：避免使用下划线开头（下划线对解释器有特殊的意义，是内建标识符使用的符号）；做到见名识义；关键字无法使用

* 运算符：包括算数运算符、**位运算符**、逻辑运算符；注意存在优先级顺序
* 赋值
 * 与C语言等不同，不需事先对变量做类型的声明，而是用一种称为“引用”的方式来实现赋值（将一个引用指向相应的内存单元）
 * 多重赋值 

            PI=pi=3.14159
 * 增量赋值

            m%=5 即 m=m%5
 * 多元赋值 引入元组的概念，而其他语言往往需第三方变量介入

             y,x=x,y 
* 语句：即完整执行一个任务的一行逻辑代码

##python数据类型
包括（长）整型、浮点型、复数型、布尔型、字符串、列表、元组、字典

* 整型
长度 32bit为-2^31-2^31-1，64bit为 -2^31-2^31-1（ python中的整型基本没有溢出的错误）
* 长整型 
整型值后加L
* 布尔型 
两个值为True和False，本质上用整型的1、0存储
* 浮点型 
即数学中的实数；可用科学计数法表示
* 复数型 
实数+虚数  note：虚数的意义（i代表逆时针旋转90度）
几个额外的函数：
    * 复数.real
    * 复数.imag
    * 复数的共轭 复数.conjugate
* 字符串
表示方式有三种： 'blah'、 "blah" 、'''blah '''，其中三引号可以指示多行字符串
* 列表 [ ]
* 元组 ( )
*字典  { } 
一种映射类型，类似于哈希表的键值对

        d={'sine':'sin','cosine':'cos','PI':3.14159}
        d['sine']
输出结果为 'sin'

##python基本运算
* 算数运算 
    * 包括正负、加减、乘除、幂、整除（//）、取余（%）等
    * 需要注意除法的运算，在spyder默认设置、不加载任何模块的环境下，3/2 输出的结果为1，3//2 输出的结果也为1，why？ 传统除法的结果与整除的结果一致，是因为**这里除法运算符左右两边的对象都是整型**
    * 所以，3.0/2的输出结果是1.5
    * 如果要3/2也能按传统除法计算，需要写命令
            
            from __future__ import division
* 比较运算 
    * 包括大于>、小于<、大于等于>=、小于等于<=、等于==、不等于!=
    * 数值之间的比较，按大小来；字符串之间的比较，按ASCII码来
    * 一个特殊例子：3<4<7，等同于C语言中的 （3<4） and  (4<7)
* 逻辑运算 
    * not,and,or
* 字符运算 
    * r指原始字符串操作符，如命令里涉及路径时，需要用r来强调路径的字符，例如
            
            f=open(r'c:\test\test.py','w')
        不加r就会报错，也可写作
            
            f=open('c:\\test\\test.py','w')
        其中“\\”意为转义字符
    * u指把字符串转成unicode的字符串
* 位运算 略
* 注意：比较运算和逻辑运算的值都是布尔型

##函数、模块和包
* **函数**是指完成一个特定功能的一段代码，包括内建函数和非内建函数，前者可直接调用，后者需导入相应的模块后再调用
    * 显示内建变量和内建函数的命令
        
            dir(builtins)
    * 查看函数帮助文档
        
            help(fuctionname)
* 如何调用非内建函数？->引入**模块**的概念，模块是指一个完整的python文件（文件指物理上的组织方式如math.py；模块指逻辑上的组织方式如math）
    * 导入模块的命令
            
            import 模块名
    * 导入多个模块 
            
            import 模块名1，模块名2
    * 导入模块里指定的模块属性 
            
            from 模块名 import 模块属性
* **包**指模块和子包组成的python应用程序环境，一个有层次的文件目录结构，形如

        包
            子包1
                模块1
                模块2
            子包2
                模块1
                模块2

    * 导入模块与导入模块属性，在使用上有一些区别，举例：调用A包下B子包下C模块下的func1函数的两种写法：
            
             import A.B.C
             A.B.C.func1(123)
             
            from A.B.C import func1
            func1(123)

* **库**指一组具有相关功能的模块的集合
* 几个概念的层级关系？
包>库/模块>函数/类
